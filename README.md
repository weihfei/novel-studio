# Novel Studio 📖

> 完整的中文网文创作系统 — 16个专业智能体，一套完整工作流。

## 解决什么问题

1. **写了就忘** → 记忆管理者维护事实数据库，每章写作前自动生成启动包
2. **文风飘移** → 文风调教师制定规范，初稿写手严格遵守
3. **逻辑漏洞** → 编辑/校对 + 润色师双重质量把关

---

## 快速开始

### 场景一：从零开始写新书

**第一步：准备项目文件**

把 `templates/` 里的模板复制到你的项目目录，填写内容：

```
templates/project.md       → 你的项目目录/project.md
templates/世界观设定.md     → 你的项目目录/世界观设定.md
templates/人物卡.md         → 你的项目目录/人物卡.md
templates/大纲规划.md       → 你的项目目录/大纲规划.md
templates/完整故事梗概.md   → 你的项目目录/完整故事梗概.md
templates/文风规范.md       → 你的项目目录/文风规范_[你的风格].md
```

**第二步：告诉 AI 初始化项目**

```
我的项目目录是 [路径]，project.md 已经填好了，请初始化记忆系统
```

AI 会自动创建 `continuity/fact_database.md` 和 `chapters/context_第1章.md`。

**第三步：开始写第1章**

```
请写第1章
```

---

### 场景二：已有设定，直接写章节

如果你已经有世界观、人物卡、大纲等文件，告诉 AI：

```
我的项目在 [路径]，project.md 路径是 [路径]，请为第N章生成启动包，然后写第N章
```

---

### 场景三：审查已有章节

```
请审查第N章，检查错别字、逻辑漏洞和设定一致性
```

## 智能体列表

**规划层**
| 智能体 | 职责 | 何时调用 |
|--------|------|----------|
| 创意顾问 | 市场分析、核心卖点 | 新书构思阶段 |
| 世界观架构师 | 世界观、力量体系、势力 | 构思阶段 |
| 角色塑造师 | 人物设计、对话风格 | 构思阶段 |
| 大纲规划师 | 故事结构、章节规划 | 构思阶段 |
| 文风调教师 | 文风规范制定 | 构思阶段 |

**执行层**
| 智能体 | 职责 | 何时调用 |
|--------|------|----------|
| 情节设计师 | 具体情节、冲突、高潮 | 写作前/卡壳时 |
| 初稿写手 | 根据启动包写正文 | 每章写作 |
| 记忆管理者 | 维护事实数据库、生成启动包 | 每章前后 |

**质量层**
| 智能体 | 职责 | 何时调用 |
|--------|------|----------|
| 编辑/校对 | 错别字、语病、逻辑漏洞 | 每章完成后（必须） |
| 润色师 | 文笔提升、表达优化 | 每章完成后（可选） |
| 过审顾问 | 平台规则合规检查 | 发布前（可选） |
| 版权顾问 | 查重、侵权风险 | 发布前（可选） |

**类型专家**
| 智能体 | 职责 | 何时调用 |
|--------|------|----------|
| 悬疑专家 | 推理逻辑、悬念铺设 | 悬疑类型 |
| 言情专家 | 情感线、CP互动 | 言情类型 |
| 科幻专家 | 硬核设定、未来科技 | 科幻类型 |
| 仙侠专家 | 修炼体系、灵宝秘境 | 仙侠类型 |

## 项目文件结构

```
你的项目目录/
├── project.md              ← 必须，所有智能体的配置入口
├── 世界观设定.md
├── 人物卡.md
├── 大纲规划.md
├── 完整故事梗概.md
├── 文风规范_[风格].md
├── continuity/
│   └── fact_database.md    ← 由记忆管理者自动维护
└── chapters/
    ├── context_第1章.md    ← 由记忆管理者自动生成
    ├── 第1章_[标题].md     ← 由初稿写手写作
    └── ...
```
